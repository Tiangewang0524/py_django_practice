2.2 数据库配置

打开testdj/settings.py。包含了 Django 项目的设置部分。

在配置数据库之前，要先设置时区和编码：

LANGUAGE_CODE = 'zh-Hans'
TIME_ZONE = 'Asia/Shanghai'

默认情况下，这个配置文件使用 SQLite 作为数据库。

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
    }
}

这其中：

ENGINE

代表使用的数据库后端，可选值有

  • 'django.db.backends.sqlite3'
  • 'django.db.backends.postgresql'
  • 'django.db.backends.mysql'
  • 'django.db.backends.oracle'

NAME

数据库的名称。如果使用的是 SQLite，数据库将是你电脑上的一个文件，在这种情况下，NAME应该是此文件的绝对路径，包括文件名。默认值为：os.path.join(BASE_DIR,'db.sqlite3')，将会把数据库文件储存在项目的根目录。

如果使用其他数据库，需要安装合适的数据库绑定组件，然后改变设置文件中DATABASES中'default'项目中的一些键值，例如采用MySQL数据库：

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'test2',
        'USER': 'root',
        'PASSWORD': '12345678',
        'HOST': 'localhost',
        'PORT': '3306',
    }
}

在已安装的mysql中创建test2数据库，字符集设置为：UTF-8

create database test2 DEFAULT CHARSET utf8;
